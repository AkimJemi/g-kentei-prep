## 個人情報保護法とGDPR

データを収集・利用するときは、利用条件・制約・法律などを確認していくことが大切になる。
- 個人情報保護法／著作権法／特許法／不正競争防止法／契約内容（商用利用可能かどうか） など

---

### 個人情報保護法

**個人情報保護法**は、個人情報の有用性に配慮しつつ、**個人の権利利益を保護**することを目的とした法律。
- 個人情報を不適切に利用されることを防ぐための法律
- 学習用データに個人情報が含まれていることがある → どのように取り扱うべきか知っておくことが大切

**対象：生存する個人**
- 亡くなった方の情報は個人情報に含まれない
- 個人情報を取り扱う場合は可能な限り**利用目的を特定**する必要がある
- 利用目的が変更する場合は、事前に本人の同意が必要になり、利用目的を本人に通知・公表しなければならない

---

### 個人情報の主要概念

| 概念名 | 定義 |
| :----- | :--- |
| **個人情報** | 氏名・生年月日・個人識別符号など、特定の個人を識別することができる情報 |
| **個人識別符号** | マイナンバー・遺伝子情報・免許証番号など、個人を識別することができる符号 |
| **個人データ** | 個人情報データベース等で管理されている個人情報（ルールに従い整理された個人情報） |
| **個人情報データベース等** | 個人情報を検索できるように体系的に構成されたデータベース |
| **保有個人データ** | 個人情報取扱事業者が開示・修正・追加・削除などを行うことができる権限を持った個人データ |

> 📌 **「個人情報」「個人データ」「保有個人データ」の違いと関係性を理解する。**

**保有個人データの例外：**
- 生命や身体・財産に危害を及ぼす恐れのあるもの（病気に関する情報など）は、保有個人データに分類されない場合がある
- 本人からの請求に応じて削除してしまうと適切な処置ができなくなるため

---

### 要配慮個人情報と機微（センシティブ）情報

- **要配慮個人情報**：人種・思想・犯罪歴など本人に対する不当な差別などが発生してしまう危険性があり、配慮すべき個人情報
  - 本人の同意なく収集したり、第三者に提供したりすることが禁止
  - **オプトアウトも不可**

- **機微（センシティブ）情報**：要配慮個人情報に政治的見解・本籍地などを含む情報
  - 金融分野における個人情報取扱事業者は、原則として取得・利用・第三者への提供が禁止

---

### オプトアウトとオプトイン

| 形式 | 説明 |
| :--- | :--- |
| **オプトアウト** | ユーザーが拒否しない限り同意しているとみなすこと |
| **オプトイン** | ユーザーが許可するまで同意していないとみなすこと |

> 📌 **要配慮個人情報はオプトアウト不可。オプトイン（本人の明示的な同意）が必要。**

---

### 匿名加工情報と仮名加工情報

| 概念名 | 定義 | 利用範囲 |
| :----- | :--- | :------- |
| **匿名加工情報** | 個人を特定できないように加工し、**元の個人情報に復元できない**ようにした情報 | 同意なく目的外利用・第三者への提供が**可能** |
| **仮名加工情報** | 他の情報と照合をしなければ個人を特定することができないように加工した情報（照合すれば復元可能） | 目的外利用は可能だが、**業務委託先を除く第三者への提供は不可** |

---

### 個人関連情報とパーソナルデータ

- **個人関連情報**：生存する個人に関する情報であって、個人情報・仮名加工情報・匿名加工情報のいずれにも該当しないもの
- **パーソナルデータ**：個人に関する情報のことで、個人情報以外にも個人の位置情報・商品の購入履歴などが含まれる

---

### GDPR（EU一般データ保護規則）

**GDPR**は、原則としてEEA（欧州経済領域：EUにアイスランド等を加えた地域）内で取得した個人データをEEA域外へ持ち出すことを禁止する法律。
- 日本は**十分性認定**を受けており、個人データを持ち出すことができる

**GDPRにおける個人情報の範囲：**
- **IPアドレス**（ネットワークに接続するパソコンなどを識別するための番号）も個人情報に含まれる
- **Cookie**（訪問したWebサイトで記入した情報などを一時的にWebブラウザに保存する仕組み）も個人情報に含まれる
- GPS位置情報も含まれる

**GDPR22条（自動処理のみに基づく判断の対象にならない権利）：**
- 利用者にとって法的効果・重大な影響を及ぼす場合、**プロファイリング**（経済状態や趣味嗜好などの個人データを分析すること）を含む自動処理のみに基づいた判断の対象にならない権利

**データポータビリティ権：**
- 企業や国が収集・蓄積した個人データを、他のサービスでも再利用できる権利

**PDS・情報銀行：**
- **パーソナルデータストア（PDS）**：個人自らが自分のデータを集約させて管理するシステム
- **情報銀行**：パーソナルデータを個人から預託され、他の事業者とのマッチングや匿名化したうえでの情報提供・一元管理を行う制度または事業者

## 著作権法

**著作権法**は、音楽・写真・小説など著作物を保護するための法律。無断で著作物を利用することが禁止されている。
- 著作物とは「思想又は感情を創作的に表現したもの」で文芸・学術・美術・音楽の範囲に属するもの
- 思想や感情を表現していない制作物などは著作物にならない

---

### 保護の対象となるもの・ならないもの

| 保護される | 保護されない |
| :--------- | :----------- |
| マニュアル、プログラムコード | AIが自動的に作成したモデル |
| 学習済みモデル（人が作成したもの） | 学習済みモデルの**パラメータ**（単なる数字、創造性なし） |
| 創造性を有するデータベース・データアセット | **ノウハウ**（経験を通して得られる知識、創造性なし） |
| 人がAIを活用して作成したもの | 単にデータを集めただけのもの |

> 📌 **AIが自動生成したものは著作物にならない。人がAIを活用して作成したものは著作物として認められる。**

---

### 著作権の種類

1. **財産権としての著作権**（**譲渡可能**）
   - **複製権**：著作物をコピーする権利
   - **翻訳権・翻案権**：翻訳したり、脚色したりする権利
   - **上映権**：著作物を上映する権利

2. **著作者人格権**（**譲渡不可**）
   - **公表権**：著作物を公表するかどうかを決定する権利
   - **氏名表示権**：本名・ペンネームどちらで表示するかを決定する権利
   - **同一性保持権**：著作物を無断で改変・切り取りなどをされない権利

> 📌 **財産権は譲渡可能。著作者人格権は著作者の精神的利益を保護するため、譲渡不可。**

---

### 共同著作物と職務著作

- **共同著作物**：2人以上の者が共同して創作した著作物で、各人の寄与分を分離して個別に利用できないもの
  - 共有者全員の同意がなければ、著作物を複製・公表・譲渡することができない

- **職務著作**：法人等の命令に基づいて、従業員が創造した著作物
  - 一定の要件を満たせば、**法人（会社）が著作者**になる
  - プログラムも要件を満たせば職務著作になる

  **職務著作の要件：**
  1. 法人等の命令に基づいて創造したもの
  2. 法人等の業務に従事する者が職務中に創造したもの
  3. 法人等が自己の著作の名義の下に公表したもの

---

### 著作権法の例外

著作権者の許諾を得ることなく複製等が可能な場合：
- 私的利用
- 教育目的での利用
- 引用の形での使用
- **データ解析での利用（著作権法30条の4）**
  - データ解析で利用する場合、著作物の複製が認められている（他の先進国は許可を取る必要がある）
  - 著作物の複製によりデータセットの作成も可能
  - 著作権者の利害に反しない場合、譲渡等（販売等）も可能

---

### 著作権の効力と保護期間

- 著作権は著作物を作った時点で**自動的に発生**する（申請不要）
- **保護期間**
  - 実名の著作物：著作者の**死後70年**を経過するまで
  - 団体名義・変名の著作物：**公表後70年**を経過するまで

---

## 特許法・不正競争防止法

### 特許法

**特許法**は、発明者の権利を守り、産業の発展を促進させる法律。
- 「自然法則を利用した技術的思想の創作のうち高度のもの」を「発明」と定義して保護
- 特許庁に出願して登録されることで権利を得る
- **保護期間：特許出願の日から20年間**

**特許権の帰属：**
- 原則として特許権は**発明者**に帰属する
- 複数人で発明した場合は、全員が発明者になる
- **職務発明**：業務中に従業員等がした発明のこと
  - 発明した従業員が特許を取得した場合、企業はその発明を実施する権利（**通常実施権**）が認められている

**実施権の種類：**

| 実施権 | 説明 |
| :----- | :--- |
| **通常実施権** | 特許発明を実施できる権利。特許権者や専用実施権者から許諾を受けるか、法律の規定で得る。独占性はない。 |
| **専用実施権** | 特許発明を**独占的に実施できる**権利。特許権者から許諾を受け、特許庁に登録が必要。A社がB社に専用実施権を設定すると、B社のみが実施でき、A社自身も実施できなくなる。 |

**出願主義：**
- 同一の発明について、最も早く特許を出願した人に特許が与えられる
- AさんとBさんがいて、Aさんが先に発明していても、Bさんが先に出願すれば、Bさんに特許が与えられる

**特許の要件：**
1. 発明であること
2. 産業上の利用可能性があること（商売が可能）
3. 新規性を有すること（社会で知られていない）
4. 進歩性を有すること（人が容易に発明できないもの）
5. 先願であること
6. 公序良俗を害する発明でないこと（人のクローン技術など）

> 📌 **特許出願後、特許として認められなかった場合でも出願内容は公開される（出願公開）。戦略的に特許出願をしないという選択肢もある。**

---

### 不正競争防止法

**不正競争防止法**は、事業者間の不正競争を防止するための法律。
- コピー商品の販売・ドメイン名の不正取得・不正な手段での**営業秘密**や**限定提供データ**の取得等が禁止
- 違反した場合は差止め請求・損害賠償請求の対象になる

**営業秘密の3要件（すべて満たす必要がある）：**

| 要件 | 説明 |
| :--- | :--- |
| **秘密管理性** | 秘密として管理されていること。従業員などから見て秘密であることが分かること（アクセス制限・社外秘などの表示）。 |
| **有用性** | 有用な営業上または技術上の情報であること。研究などの失敗データも有用性が認められる。公序良俗に反する情報（公害情報・脱税情報など）は対象外。 |
| **非公知性** | 公然と知られていないこと。保有者の管理下以外では一般的に入手できない状態。一般的に購入できる書籍や誰でも閲覧できるホームページに記載の情報は非公知性が認められない。 |

> 📌 **3つの要件を満たしていない情報は営業秘密ではないので、保護の対象にならない。**

**限定提供データ：**
- 業として**限定された人**に提供する、電磁的方法で記録・管理されている相当量蓄積された技術上・営業上のデータ
- 「限定された人」＝条件を満たせばデータにアクセスできる者（月額料金等を支払ってアクセス可能な者など）
- 限定された人のみにデータを提供しているという意思を第三者が認識できるようにする必要がある（パスワードなど）
- **保護対象外のケース：**
  - 無償で不特定多数の人に公開されている情報
  - 不正に関するデータ（技術上・営業上のデータに該当しない）
  - 秘密として管理されているデータ → **営業秘密**として保護されるため限定提供データには該当しない
  - 月額料金を支払えば誰でもアクセスできる場合 → 秘密として管理されていないと判断され、営業秘密には該当しない

## 知的財産権

**知的財産権**は、創造活動によって生み出された創造物について、その創出者に対して付与される権利。

**知的財産権を保護する主な法律：**

| 法律 | 保護対象 |
| :--- | :------- |
| 著作権法 | 文芸・学術・美術・音楽などの著作物 |
| 特許法 | 「発明」（高度な技術的思想の創作） |
| 不正競争防止法 | 営業秘密・限定提供データ |
| **商標法** | 商品やサービスのロゴやマーク（商標） |
| **意匠法** | 工業デザイン（製品の形状・模様・色彩など） |
| **実用新案法** | 物品の形状・構造・組合せに係る考案（特許法より簡易な審査） |

**産業財産権：**特許権・実用新案権・意匠権・商標権の4つの権利。特許庁が所管。

---

### AI分野における知的財産権の保護

- **営業秘密として保護**：AIのアルゴリズム・学習データ・学習済みモデルを営業秘密として管理
  - データを暗号化などを行い**リバースエンジニアリング**（分解・解析を行い、仕様などを調査すること）を防ぐ

- **秘密保持契約（NDA）**：企業同士などで結ばれる、特定の情報を第三者に公開しないことを約束する契約
  - 自分たちの権利を守るためにも契約を結ぶことが大切

---

## 独占禁止法

**独占禁止法**は、私的独占・不当な取引制限・不公正な取引を禁止し、事業支配力の過度な集中を防止して、事業者間の公正かつ自由な競争を促進し、消費者の利益を確保することを目的とする法律。

| 規制内容 | 説明 |
| :------- | :--- |
| **私的独占** | 事業者が他の事業者の活動を排除・支配することで、一定の取引分野における競争を実質的に制限すること |
| **不当な取引制限** | 事業者間の協定等により価格や生産量を決定し、競争を制限すること（カルテル・談合など） |
| **不公正な取引** | 正当な理由なく競争者と共同して、他の事業者の事業活動を困難にさせるおそれがあること |
| **事業支配力の過度な集中** | 買収・合併等により市場を実質的に支配してしまうこと |
| **競争制限** | カルテルなどにより、特定の事業者等によって市場が支配され、競争が実質的に制限される状態 |
| **公正競争阻害性** | 公正な競争に悪影響を及ぼすおそれのあること |

> 📌 **カルテル：競争回避のため複数事業者間で価格などを決めること。独占状態（1社）は消費者に不利、寡占状態（複数社）は価格が下がりにくい特徴がある。**

## AI開発委託契約

**契約自由の原則：**公序良俗に反する場合など以外は、契約は当事者間で、国家に干渉されず、自由に結ぶことができるという原則。

### 主な契約形態

| 契約形態 | 内容 |
| :------- | :--- |
| **請負契約** | 請負人が仕事を完成させることを約束し、注文者がその仕事の結果に対して報酬を支払う契約 |
| **準委任契約** | 受託者に事務処理（法律行為以外）を委託し、業務の遂行等に応じて報酬を支払う契約 |
| **委任契約** | 受託者に法律行為を委託し、業務の遂行等に応じて報酬を支払う契約 |

> 📌 **AI開発では、精度を保証することが難しいため、仕事を完成させられるとは限らない → 準委任契約が多い。**

---

### AI・データの利用に関する契約ガイドライン（経済産業省）

データ契約に関して実務の実績が少ないことからトラブルが発生しやすいため、経済産業省が作成。

**開発フェーズと契約：**

| フェーズ | 内容 | 締結する契約 |
| :------- | :--- | :----------- |
| **アセスメント** | サンプルデータを受領し、AI導入が可能かどうか検証する段階 | 秘密保持契約等 |
| **PoC（概念実証）** | AIモデルを実際に作成し、実務でも使用できるか検証する段階 | 導入検証契約書等 |
| **開発** | 実務で使用するモデルを作成する段階 | ソフトウェア開発契約書等 |
| **追加学習** | モデル運用後に精度向上等のために追加的に学習させる段階 | 多様な契約（新規開発契約書・保守契約への追加等） |

## AIサービス提供契約とデータ契約

### SaaS（Software as a Service）

インターネット経由でソフトウェアをサービスとして提供する形態。
- Gmail・Slack・Dropbox・Zoomなど様々なサービスがある
- 月額サービスとして提供していることが多い
- 利用者側は自社で開発・運用する必要がないという利点がある
- 多くのサービス内にAIを使用した機能が追加されている（迷惑メール振り分けなど）
- SaaS提供企業は多くのユーザーの情報を取得する → 提供企業はユーザーの情報をどこまで利用可能か、ユーザーは提供されたデータをどこまで利用可能か定める必要がある
- 提供しているAIの精度やサービスの品質（稼働率等）をユーザーに報告することも大切（**精度保証**）

---

### データ契約の3類型

AI・データの利用に関する契約ガイドライン（経済産業省）に記載された3つの契約類型：

| 契約類型 | 説明 |
| :------- | :--- |
| **データ提供型契約** | データ提供者のみが保有しているデータを他の者に提供するときに、データ提供の条件などを取り決める契約 |
| **データ創出型契約** | 複数当事者が関与することによって新たにデータが創出される場面で、データの利用権限などを取り決める契約 |
| **データ共有（プラットフォーム型）契約** | 複数事業者がプラットフォーム（PF）にデータを提供し、PFが加工等したデータを共有するときに取り決める契約 |
