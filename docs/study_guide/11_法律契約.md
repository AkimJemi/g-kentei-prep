# AIに関する法律と契約

> **G検定 学習ガイド** | 分野：法律契約 | 95P | 2026-02-25

---

AI技術の開発や利用を進める上で、データは不可欠な要素です。しかし、そのデータを収集し、利用する際には、さまざまな法律や契約上の制約を理解し、遵守することが極めて重要になります。特に、**個人情報保護法**、**著作権法**、**特許法**、**不正競争防止法**、そして**独占禁止法**といった法律は、AIプロジェクトの成否を左右する可能性があり、G検定でも頻繁に問われる分野です。

本章では、これらの法律や関連する概念について、AI開発・利用の視点からわかりやすく解説していきます。

## 個人情報保護法とGDPR

AIの学習データには、しばしば個人の情報が含まれることがあります。これらの情報を適切に取り扱うことは、個人の権利利益を保護し、社会からの信頼を得る上で不可欠です。日本では**個人情報保護法**が、EU（欧州連合）では**GDPR（一般データ保護規則）**が、個人データの保護に関する主要な法律として機能しています。

### 個人情報保護法の目的と対象

**個人情報保護法**は、**個人情報の有用性**に配慮しつつ、**個人の権利利益を保護**することを目的とした法律です。個人情報が不適切に利用されることを防ぎ、安心してデータが活用される社会を目指しています。

この法律の対象は「生存する個人」の情報であり、**亡くなった方の情報は個人情報には含まれません**。個人情報を取り扱う事業者は、可能な限り**利用目的を特定**し、その目的の範囲内で情報を利用しなければなりません。利用目的を変更する場合は、原則として事前に本人の同意を得るか、本人に通知または公表する必要があります。

> 📌 **G検定頻出ポイント**:
> *   個人情報保護法の目的は「個人情報の有用性に配慮しつつ、個人の権利利益を保護すること」。
> *   対象は「生存する個人」であり、故人の情報は含まれない。
> *   個人情報の利用目的の特定と、目的外利用の制限が重要。

### 個人情報保護法における主要な概念

個人情報保護法では、情報の種類に応じて様々な定義がされています。これらの違いを理解することは、適切なデータ管理を行う上で不可欠です。

| 概念名 | 定義 | 具体例 |
| :----- | :--- | :----- |
| **個人情報** | 生存する個人に関する情報で、特定の個人を識別できるもの（他の情報と容易に照合できる場合を含む）。 | 氏名、生年月日、住所、電話番号、メールアドレス、顔写真、指紋、**個人識別符号** |
| **個人識別符号** | 特定の個人の身体の一部の特徴をデータ化したものや、サービス利用や書類に付与される符号で、政令で定められたもの。 | マイナンバー、運転免許証番号、パスポート番号、指紋データ、顔認識データ、DNA情報、声紋データ |
| **個人データ** | **個人情報データベース等**を構成する個人情報。つまり、ルールに従って整理され、検索可能な状態にある個人情報。 | 顧客リスト（氏名、住所、電話番号が紐付けられ、検索可能な状態） |
| **個人情報データベース等** | 個人情報を含む情報の集合体で、特定の個人情報を検索できるように体系的に構成されたもの。 | 顧客管理システム、会員データベース、人事データベース |
| **保有個人データ** | **個人情報取扱事業者**が、開示、訂正、追加、削除、利用停止などの権限を持つ個人データ。本人からの請求に応じて、これらの対応を行う義務がある。 | 企業が管理する顧客データのうち、開示・削除等の請求に対応できるもの |

> 📌 **G検定頻出ポイント**:
> *   「個人情報」「個人データ」「保有個人データ」の違いと関係性を理解する。
> *   **個人識別符号**の具体例を把握する。
> *   **保有個人データ**は、原則として本人からの開示・削除等の請求に応じる義務がある。

#### 保有個人データに関する例外

ただし、**保有個人データ**であっても、生命、身体、財産に危害を及ぼす恐れがある情報などは、本人からの請求に応じて削除などを行う必要がない場合があります。例えば、病気に関する情報は、適切な処置のために削除できないケースが該当します。

### 要配慮個人情報と機微（センシティブ）情報

特に慎重な取り扱いが求められる個人情報もあります。

*   **要配慮個人情報**:
    人種、信条、社会的身分、病歴、犯罪の経歴、犯罪被害に関する情報など、不当な差別や偏見が生じる可能性のある個人情報。原則として、**本人の同意なく収集したり、第三者に提供したりすることは禁止**されており、後述する**オプトアウト**も認められません。

*   **機微（センシティブ）情報**:
    **要配慮個人情報**に加えて、政治的見解、本籍地、労働組合への加盟、性生活に関する情報などを含む、より広範な概念です。特に金融分野における個人情報取扱事業者は、原則として**機微（センシティブ）情報**の取得、利用、第三者への提供が禁止されています。

> 📌 **G検定頻出ポイント**:
> *   **要配慮個人情報**は、本人の同意なしの収集・第三者提供が原則禁止。
> *   **機微（センシティブ）情報**は、金融分野で特に厳しく規制される。

### オプトアウトとオプトイン

個人情報の利用に関する同意の形式には、以下の2種類があります。

| 形式名 | 定義 | 特徴 |
| :----- | :--- | :--- |
| **オプトアウト** | ユーザーが拒否の意思表示をしない限り、同意しているとみなす方式。 | ユーザーが意識的に拒否しない限り、情報利用が進む。 |
| **オプトイン** | ユーザーが明示的に許可するまで、同意していないとみなす方式。 | ユーザーの積極的な同意が必要。より厳格な同意形式。 |

**要配慮個人情報**の第三者提供には、**オプトアウト**は認められず、原則として**オプトイン**（本人の明示的な同意）が必要です。

### 匿名加工情報と仮名加工情報

個人情報を加工することで、利用の自由度を高めるための制度です。

| 概念名 | 定義 | 特徴 | 利用範囲 |
| :----- | :--- | :--- | :------- |
| **匿名加工情報** | 個人を特定できないように個人情報を加工し、**元の個人情報に復元できない**ようにした情報。 | 個人を特定できる情報を削除したり、復元できない文字に置き換えたりする。 | **同意なく目的外利用・第三者への提供が可能**。 |
| **仮名加工情報** | 他の情報と照合をしなければ個人を特定することができないように加工した情報。**他の情報と照合することで個人の特定が可能**。 | 氏名などを削除し、識別子を付与するなど。 | **本人による同意がなく、目的外の利用はできる**が、**業務委託先を除く第三者への提供はできない**。 |

> 📌 **G検定頻出ポイント**:
> *   **匿名加工情報**は復元不可、**仮名加工情報**は照合すれば復元可能。
> *   それぞれの情報が、同意なしの目的外利用や第三者提供がどこまで許されるかを理解する。

### 個人関連情報とパーソナルデータ

*   **個人関連情報**:
    生存する個人に関する情報であって、個人情報、仮名加工情報、匿名加工情報のいずれにも該当しないもの。例えば、Webサイトの閲覧履歴や位置情報など、単体では個人を特定できないが、他の情報と組み合わせることで個人情報になり得るデータです。

*   **パーソナルデータ**:
    個人に関する情報の総称で、個人情報だけでなく、個人の位置情報、商品の購入履歴、Webサイトの閲覧履歴など、広範なデータが含まれます。

### GDPR（EU一般データ保護規則）

**GDPR**は、EUおよびEEA（欧州経済領域：EU加盟国にアイスランド、リヒテンシュタイン、ノルウェーを加えた地域）における個人データの保護を目的とした法律です。

*   **目的と対象**:
    原則としてEEA内で取得した個人データをEEA域外へ持ち出すことを禁止しています。しかし、日本は「個人データについて保護水準を満たす国」だとする**十分性認定**を受けているため、EEAから日本へ個人データを持ち出すことが可能です。

*   **GDPRにおける個人情報の範囲**:
    GDPRでは、**IPアドレス**（ネットワークに接続するパソコンなどを識別するための番号）や**Cookie**（Webサイトの訪問履歴などを一時的にWebブラウザに保存する仕組み）、GPSの位置情報なども**個人情報**に含まれるとされています。これは、これらの情報が個人を特定し得る情報とみなされるためです。

*   **GDPR22条（自動処理のみに基づく判断の対象にならない権利）**:
    利用者にとって法的効果や重大な影響を及ぼす場合、**プロファイリング**（経済状態や趣味嗜好などの個人データを分析すること）を含む自動処理のみに基づいた判断の対象にならない権利が規定されています。これは、AIによる自動的な意思決定が個人の権利を侵害しないようにするためのものです。

*   **データポータビリティ権**:
    企業や国が収集・蓄積した個人データを、他のサービスでも再利用できる権利のことです。これにより、ユーザーは自分のデータをより自由に管理・活用できるようになります。

*   **パーソナルデータストア（PDS）と情報銀行**:
    **パーソナルデータストア（PDS）**は、個人自らが自分のデータを集約させて管理するシステムです。
    **情報銀行**は、パーソナルデータを個人から預託され、他の事業者とのマッチングや匿名化した上での情報提供、一元管理を行う制度または事業者のことを指します。

> 📌 **G検定頻出ポイント**:
> *   GDPRの適用範囲（EEA）と日本の**十分性認定**。
> *   GDPRでは**IPアドレス**や**Cookie**も個人情報に含まれる。
> *   **データポータビリティ権**と**GDPR22条**の自動処理に関する権利。
> *   **情報銀行**の役割。

## 著作権法

AI開発において、学習データの収集やAIが生成したコンテンツの取り扱いは、**著作権法**と密接に関わってきます。著作権法は、音楽、写真、小説、プログラムコードなど、**著作物**を保護するための法律です。

### 著作権法の目的と著作物

**著作権法**は、著作者の権利を保護し、文化の発展に寄与することを目的としています。

*   **著作物**とは:
    「思想又は感情を創作的に表現したものであって、文芸、学術、美術又は音楽の範囲に属するもの」と定義されています。
    つまり、単なる事実やデータ、アイデアそのものは著作物にはなりません。**創造性**と**表現**が重要です。

### 保護の対象となるもの・ならないもの

AI開発に関連して、何が著作物として保護されるのか、されないのかを理解することは非常に重要です。

| 保護の対象となるもの | 保護の対象とならないもの |
| :------------------- | :----------------------- |
| マニュアル、プログラムコード | AIが自動的に作成したモデル（人が作成したもののみ保護対象） |
| 学習済みモデル（人が作成したもの） | 学習済みモデルのパラメータ（単なる数字であり、創造性がない） |
| 創造性を有するデータベース・データアセット（情報の選択や体系的な構成に創造性がある場合） | ノウハウ（経験を通して得られる知識であり、創造性がない） |
| 人がAIを活用して作成したもの | 単にデータを集めただけのもの（創造性がない場合） |

> 📌 **G検定頻出ポイント**:
> *   **AIが自動生成したものは原則として著作物にならない**。
> *   **学習済みモデルのパラメータ**や**ノウハウ**は著作物として保護されない。
> *   **人がAIを活用して作成したものは著作物として認められる**可能性がある。

### 著作権の種類

著作権は、大きく分けて2つの権利から構成されます。

1.  **財産権としての著作権（著作権）**:
    著作物の利用に関する経済的な利益を保護する権利です。これは**譲渡可能**です。
    *   **複製権**: 著作物をコピーする権利。
    *   **翻訳権・翻案権等**: 著作物を翻訳したり、脚色したりする権利。
    *   **上映権**: 著作物を上映する権利。
    *   その他、公衆送信権、展示権など多岐にわたります。

2.  **著作者人格権**:
    著作者の精神的利益を保護する権利です。これは著作者に一身専属のものであり、**譲渡することはできません**。
    *   **公表権**: 著作物を公表するかどうか、いつ、どのような方法で公表するかを決定する権利。
    *   **氏名表示権**: 著作者名を表示するかどうか、本名かペンネームかなどを決定する権利。
    *   **同一性保持権**: 著作物を無断で改変、切除、またはその他の変更をされない権利。

> 📌 **G検定頻出ポイント**:
> *   **財産権としての著作権は譲渡可能**だが、**著作者人格権は譲渡できない**。
> *   著作者人格権の3つの権利（公表権、氏名表示権、同一性保持権）を理解する。

### 共同著作物と職務著作

*   **共同著作物**:
    2人以上の者が共同して創作した著作物で、各人の寄与分を分離して個別に利用できないものです。この場合、共有者全員の同意がなければ、著作物の複製や公表、譲渡を行うことはできません。

*   **職務著作**:
    法人（会社）などの命令に基づいて、従業員が職務中に創作した著作物で、一定の要件を満たせば、著作物を創作した自然人（従業員）ではなく、**法人（会社）が著作者となる**制度です。プログラムも要件を満たせば職務著作になります。

    **職務著作の要件**:
    1.  法人等の発意に基づき創作されたものであること。
    2.  法人等の業務に従事する者が職務上作成したものであること。
    3.  法人等が自己の著作の名義の下に公表するものであること（ただし、公表されない著作物については、作成の目的および態様から法人等が著作者であると認められる場合）。

### 著作権法の例外

著作権は著作者を保護する重要な権利ですが、過度な保護は文化の発展を阻害する可能性があります。そのため、著作権者の許諾を得ることなく著作物を利用できる例外規定が設けられています。

*   **私的利用**: 個人または家庭内など、限られた範囲内で利用する場合。
*   **教育目的での利用**: 学校などの教育機関での利用。
*   **引用**: 公正な慣行に合致し、報道、批評、研究その他の目的上正当な範囲内で行われる場合。
*   **データ解析での利用**:
    **著作権法30条の4**では、情報解析の用に供する場合、著作物の複製が認められています。これは、AIの学習データとして著作物を利用する際に、原則として著作権者の許可を取る必要がないことを意味します。
    さらに、著作物の複製によりデータセットを作成することも可能であり、著作権者の利益を不当に害しない限り、そのデータセットの譲渡（販売など）も可能です。

> 📌 **G検定頻出ポイント**:
> *   **著作権法30条の4**により、**データ解析目的での著作物の複製は原則として許可不要**。
> *   データセットの作成や、著作権者の利益を害しない範囲での譲渡も可能。

### 著作権の効力と保護期間

*   **著作権の発生**:
    著作権は、著作物を作った時点で**自動的に発生**します（無方式主義）。権利を保護してもらうための申請を行う必要はありません。

*   **保護期間**:
    *   **実名の著作物**: 著作者の死後70年を経過するまで。
    *   **団体名義の著作物（法人著作）や変名（ペンネームなど）の著作物**: 公表後70年を経過するまで。

## 特許法・不正競争防止法

AI技術そのものや、AIを活用したビジネスモデル、あるいはAI開発で得られたノウハウやデータは、**特許法**や**不正競争防止法**によって保護される可能性があります。

### 特許法

**特許法**は、発明者の権利を保護し、産業の発展を促進させることを目的とした法律です。

*   **発明の定義**:
    「自然法則を利用した技術的思想の創作のうち高度のもの」と定義されています。

*   **権利の取得と保護期間**:
    特許庁に出願し、審査を経て登録されることで**特許権**という独占的な権利を得ます。保護期間は**特許出願の日から20年間**です。

*   **特許権の帰属**:
    原則として特許権は**発明者**に帰属します。複数人で発明した場合は、全員が発明者となります。

*   **職務発明**:
    業務中に従業員等がした発明を**職務発明**といいます。職務発明において、発明した従業員が特許を取得した場合、企業はその発明を実施する権利（**通常実施権**）が認められています。

#### 実施権の種類

**実施権**とは、特許を所有する者から妨げられることなく、特許発明を実施できる権利のことです。実施とは、特許対象の物を生産したり、使用したりすることなどを指します。

| 実施権の種類 | 定義 | 特徴 |
| :----------- | :--- | :--- |
| **通常実施権** | 特許発明を実施できる権利。特許権者や専用実施権者から許諾を受けるか、法律の規定により発生する。 | 法律の規定や契約で定めた範囲内において、商売としてその特許発明の実施をする権利を持つ。**独占性はない**。 |
| **専用実施権** | 特許発明を**独占的に実施できる権利**。特許権者から許諾を受け、特許庁に登録する必要がある。 | 設定された範囲内では、特許権者自身もその発明を実施できなくなる。**独占性がある**。 |

> 📌 **G検定頻出ポイント**:
> *   **職務発明**の場合、企業は**通常実施権**を持つ。
> *   **通常実施権**と**専用実施権**の違い（特に独占性の有無）。

#### 出願主義

特許法では、同一の発明について、最も早く特許を出願した人に特許を与えるという**出願主義**を採用しています。
例えば、Aさんが先に発明していても、Bさんが先に特許を出願したら、Bさんに特許が与えられます。

#### 特許の要件

特許として認められるためには、以下の要件を満たす必要があります。

*   **発明であること**: 自然法則を利用した技術的思想の創作であること。
*   **産業上の利用可能性があること**: 商売として利用できること。
*   **新規性を有すること**: 社会で知られていない、新しい発明であること。
*   **進歩性を有すること**: その分野の専門家が容易に発明できない、高度な発明であること。
*   **先願であること**: 他の誰よりも早く出願していること。
*   **公序良俗を害する発明でないこと**: 人のクローン技術など、社会の秩序や善良な風俗に反しないこと。

#### 特許出願公開のデメリット

特許出願後、特許として認められなかった場合でも、出願した内容は原則として**公開されてしまいます（出願公開）**。これにより、誰でもその内容を見ることができ、技術的な優位性を失う可能性があります。そのため、特許の要件を満たしていたとしても、戦略的に特許出願をせず、後述する**営業秘密**として管理するという選択肢を取ることもあります。

### 不正競争防止法

**不正競争防止法**は、事業者間の公正かつ自由な競争を促進するため、不正な競争行為を防止する法律です。

*   **目的と禁止行為**:
    コピー商品の販売、ドメイン名の不正取得、そして**不正な手段での営業秘密や限定提供データの取得・利用・開示**などが禁止されています。これらの行為を行った場合、差止め請求や損害賠償請求の対象となります。

#### 営業秘密

**営業秘密**とは、企業が持つ秘密情報のうち、以下の3つの要件をすべて満たすものです。製造に関するノウハウや顧客情報などが該当します。

1.  **秘密管理性**:
    秘密として管理されていること。従業員などから見て、それが秘密であることが分かるように、アクセス制限（パスワード設定など）や「社外秘」などの表示がされている必要があります。

2.  **有用性**:
    有用な営業上または技術上の情報であること。研究開発の失敗データなども有用性が認められることがあります。ただし、公序良俗に反する内容の情報（公害に関する情報、脱税に関する情報など）は保護の対象になりません。

3.  **非公知性**:
    公然と知られていないこと。保有者の管理下以外では一般的に入手できない状態である必要があります。一般的に購入できる書籍や誰でも閲覧できるホームページに記載されている情報は、非公知性が認められません。

> 📌 **G検定頻出ポイント**:
> *   **営業秘密の3要件**（秘密管理性、有用性、非公知性）を正確に理解する。
> *   これらの要件を満たさない情報は、営業秘密として保護されない。

#### 限定提供データ

**限定提供データ**とは、業として限定された人に提供する、電磁的方法で記録・管理されている相当量蓄積された技術上・営業上のデータのことです。

*   **要件**:
    *   業として限定された人に提供されていること（例：月額料金を支払ってアクセスできる会員制データ）。
    *   電磁的方法で記録・管理されていること。
    *   相当量蓄積されていること。
    *   技術上または営業上のデータであること。

*   **保護対象外となるケース**:
    *   無償で不特定多数の人に公開されている情報。
    *   不正に関するデータなど、技術上・営業上のデータに該当しないもの。
    *   秘密として管理されているデータは**営業秘密**として保護されるため、限定提供データには該当しません。
    *   月額料金等を支払ってデータを閲覧できる場合でも、閲覧のためにパスワード等が設定されていたとしても、条件を満たせば誰でもデータにアクセスできるため、秘密として管理されていないと判断され、営業秘密には該当しません。

> 📌 **G検定頻出ポイント**:
> *   **限定提供データ**の定義と、**営業秘密**との違いを理解する。
> *   「限定された人」への提供意思を第三者が認識できるようにする必要がある（例：パスワード保護）。

## 1. 独占禁止法とAI・データビジネス

AIやデータ活用が進む現代において、特定の企業が市場を不当に支配したり、競争を阻害したりする行為は、健全な経済活動や消費者の利益を損なう可能性があります。これを防ぐために存在する法律が**独占禁止法**です。

### 1.1. 独占禁止法の目的と規制対象

独占禁止法は、公正かつ自由な競争を促進し、消費者の利益を保護することを目的としています。AI・データビジネスにおいても、以下のような行為が規制の対象となります。

#### 不公正な取引

正当な理由なく、競争者と共同して、他の事業者に対して商品の供給数量などを制限したり、取引を妨害したりすることで、その事業活動を困難にさせるおそれがある行為を指します。

> 📌 **G検定頻出ポイント**:
> 不公正な取引は、市場における自由な競争を阻害し、特定の事業者が不当な優位性を得ることを防ぐための規制です。AI関連サービスやデータの提供においても、特定の企業が不当に市場を支配するような行為は規制対象となり得ます。

#### 事業支配力の過度な集中

事業者が他の事業者を買収（M&A）したり合併したりすることで、市場を実質的に支配してしまう状態を指します。

*   **独占状態（1社）**: 1社が市場を完全に支配すると、競争が行われなくなり、価格が高止まりしたり、サービスの質が低下したりするなど、消費者の利益に反するおそれがあります。
*   **寡占状態（複数社）**: 数社が市場の大部分を占める状態でも、企業間の競争が限定的になり、価格が下がりにくいといった特徴があります。

AI技術や大量のデータを保有する企業が、関連するスタートアップ企業を次々と買収し、市場を独占するようなケースは、この規制の対象となる可能性があります。

#### 競争制限

**カルテル**（企業間の価格協定など）や**談合**（入札における企業間の調整）などにより、特定の事業者によって市場が支配され、競争が実質的に制限される状態を指します。これは、消費者がより良い製品やサービスを、より安い価格で享受する機会を奪うことになります。

#### 公正競争阻害性

上記のような行為によって、公正な競争に悪影響を及ぼすおそれがあることを指します。独占禁止法は、実際に悪影響が発生していなくても、「おそれ」がある段階で規制の対象とすることがあります。
