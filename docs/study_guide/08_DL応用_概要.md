# 08 DL応用 ── 概要・重要用語集

> **G検定 クイックリファレンス** | 分野：DL応用 | 2026-02-25

## 📖 章の概要

CNN応用（画像認識・物体検出・セグメンテーション）、NLP応用、生成AI、マルチモーダルAI、説明可能AI（XAI）、モデル軽量化などを扱う章。

---

## 🗂️ 大項目一覧と要点

### 画像認識・物体検出
| 用語 | 内容 |
|---|---|
| **画像分類** | 画像全体のカテゴリを予測 |
| **物体検出** | 物体の位置（バウンディングボックス）とクラスを同時予測 |
| **セグメンテーション** | ピクセルレベルでのクラス分類 |
| **インスタンスセグメンテーション** | 同クラスの複数物体を個別に識別 |
| **姿勢推定** | 関節点の位置を推定して人体の姿勢を認識 |

### 主要CNNアーキテクチャ
| モデル | 特徴 |
|---|---|
| **AlexNet** | ILSVRC 2012優勝。ディープラーニング普及の契機（2012） |
| **VGGNet** | 3×3の小さいフィルタを重ねる単純な構造（2014） |
| **GoogLeNet（Inception）** | Inceptionモジュール。計算効率化（2014） |
| **ResNet** | 残差接続（スキップ接続）で100層超えを実現（2015） |
| **EfficientNet** | スケーリング法則で精度・効率を両立 |

### 物体検出アーキテクチャ
| モデル | 特徴 |
|---|---|
| **R-CNN系（Faster R-CNN）** | 候補領域を先に提案してから分類 |
| **YOLO** | 1パスで位置とクラスを同時予測。高速 |
| **SSD** | 複数スケールで検出。YOLO系の改善版 |

### 自然言語処理（NLP）応用
| タスク | 内容 |
|---|---|
| **感情分析** | テキストの感情（ポジティブ・ネガティブ）を判定 |
| **固有表現認識（NER）** | 人名・地名・組織名などをテキストから抽出 |
| **機械翻訳** | 一言語から別言語への自動翻訳 |
| **質問応答（QA）** | 文脈と質問から回答を生成 |
| **文章要約** | 長文を短い要約文に変換 |

### 生成AIと高度生成モデル
| モデル | 特徴 |
|---|---|
| **拡散モデル** | 拡散過程（ノイズ付与）→逆拡散（ノイズ除去）で生成 |
| **VAE** | エンコーダ→確率分布サンプリング→デコーダ。多様な生成 |
| **GAN** | Generator（偽データ生成）vs Discriminator（識別）の対立学習 |
| **Pix2Pix** | 条件付きGAN（CGAN）。ペア画像が必要 |
| **Cycle GAN** | ペア画像不要のCGAN。2ドメイン相互変換 |
| **NeRF** | 複数の2D画像から3Dシーンを生成する技術 |
| **Dall-E** | CLIPと拡散モデルを組み合わせたText-to-Image |

### マルチモーダルAI
| 用語 | 内容 |
|---|---|
| **マルチモーダルAI** | 複数の入力形式（テキスト・画像・音声等）を処理するAI |
| **CLIP** | 画像とテキストのペア学習。Zero Shot Learningが可能 |
| **Zero Shot Learning** | 未学習のクラスでもテキスト情報で対応できる能力 |
| **VQA（Visual Question Answering）** | 画像と質問から回答を生成するタスク |
| **一気通貫学習** | ロボットの一連の動作を1つのネットワークで実現 |
| **基盤モデル** | 大規模事前学習した汎用モデル（ファインチューニングで特化） |

### 説明可能AI（XAI）
| 手法 | 内容 |
|---|---|
| **LIME** | 局所的な線形モデルで予測の根拠を近似 |
| **SHAP** | Shapley値で各特徴量の貢献度を計算 |
| **Permutation Importance** | 特徴量をシャッフルして重要度を測定 |
| **CAM** | CNNが画像のどこを重視したかをヒートマップで可視化 |
| **Grad-CAM** | 勾配情報を使い、より汎用的にCAMを実現 |

### モデル軽量化・エッジAI
| 用語 | 内容 |
|---|---|
| **エッジAI** | エッジデバイス上で推論。低遅延・高プライバシー |
| **クラウドAI** | クラウド上で学習・推論。大規模処理に強い |
| **蒸留（Distillation）** | 教師モデル→生徒モデルへ知識転移（soft target使用） |
| **量子化** | 重みのビット数削減（32bit→8bit等）。計算量削減 |
| **プルーニング** | 重要度の低い重み接続を削除 |
| **宝くじ仮説** | 大きなネットワーク内に小さな高性能サブネットが存在する |
