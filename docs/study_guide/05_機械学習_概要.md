# 05 機械学習 ── 概要・重要用語集

## 📖 章の概要

機械学習の4種類の学習方法から始まり、各アルゴリズム（線形回帰・SVM・決定木等）、
クラスタリング・次元削減、強化学習、モデル評価（交差検証・混同行列・F値等）、
過学習対策（正則化・ノルム）まで網羅的に扱う章。

---

## 🗂️ 学習の種類

| 種類 | 特徴 |
|---|---|
| **教師あり学習** | 特徴量**と**正解ラベル付きデータで学習。分類・回帰問題に使用 |
| **教師なし学習** | 正解ラベルなし。クラスタリング・次元削減に使用 |
| **半教師あり学習** | 少量の正解ラベル付き＋大量のラベルなしデータを活用。教師あり学習より一般的に精度は低い |
| **強化学習** | エージェントが報酬を最大化するよう試行錯誤。お掃除ロボット・囲碁など |

- **分類問題**：データが属するカテゴリ（離散値）を予測
- **回帰問題**：連続値（体重・温度など）を予測
- 半教師あり学習の代表手法：**FixMatch**（擬似ラベリング＋一致性正則化）

## 🗂️ 教師あり学習：主要アルゴリズム

### 線形回帰
| 用語 | 内容 |
|---|---|
| **単回帰分析** | 1つの目的変数を1つの説明変数で予測 |
| **重回帰分析** | 1つの目的変数を複数の説明変数で予測 |
| **目的変数** | 予測したい変数（結果） |
| **説明変数** | 目的変数を説明するための変数（原因） |
| **偏回帰係数** | 重回帰で各説明変数が予測に与える影響の大きさ |
| **多重共線性** | 相関の高い説明変数が複数存在→計算が不安定・精度低下 |
| **最小二乗法** | 実際の値と予測値の差の2乗和が最小になるパラメータを求める方法 |
| **ラッソ回帰（Lasso）** | L1正則化を適用した線形回帰（特徴選択効果あり） |
| **リッジ回帰（Ridge）** | L2正則化を適用した線形回帰（重みを小さく保つ） |
| **Elastic Net** | ラッソ回帰とリッジ回帰を組み合わせたもの |

### ロジスティック回帰
- ある事象が発生する確率を予測する手法
- 2値分類：**シグモイド関数**（出力0〜1）
- 多クラス分類：**ソフトマックス関数**（合計=1）
- 閾値（デフォルト0.5）を変えることで適合率・再現率を調整可能

### 決定木
| 用語 | 内容 |
|---|---|
| **情報利得** | 分割前の不純度 − 分割後の不純度。最大化するよう分割 |
| **不純度** | 1つのノードに異なるクラスのサンプルがどれだけ混在するか |
| **ジニ係数** | 不純度を測る指標の1つ |
| **エントロピー** | 不純度を測る指標の1つ（情報理論から）|
| **根ノード（ルートノード）** | 木の最上位のノード |
| **葉ノード（リーフノード）** | 末端のノード（予測結果が入る）|
| **回帰木** | 回帰で使用される決定木 |
| **分類木** | 分類で使用される決定木 |
| 特徴 | 解釈性が高い。ただし分岐が多いと過学習しやすい |

### ランダムフォレスト
| 用語 | 内容 |
|---|---|
| **ランダムサンプリング** | 無作為に一部の訓練データを抽出すること |
| **ブートストラップサンプリング** | **重複を許して**無作為に一部のデータを抽出すること |
| **復元抽出** | 抽出後に元に戻してから次を抽出 |
| **非復元抽出** | 抽出後に元に戻さずに次を抽出 |
| バギングの一種 | 決定木を並列に複数作成して多数決（分類）または平均（回帰）|
| 特徴 | 決定木の数が増えても過学習しにくい |

### アンサンブル学習
| 手法 | 特徴 |
|---|---|
| **バギング** | 複数モデルを**並列**学習・多数決。ランダムフォレストが代表。学習時間短 |
| **ブースティング** | モデルを**直列**学習・前のモデルの誤りを修正。過学習になりやすい。計算コスト高 |
| **勾配ブースティング** | 勾配降下法を用いたブースティング |
| **XGBoost** | 決定木と勾配ブースティングを組み合わせた手法 |
| **AdaBoost** | 誤分類したデータの重みを大きくして精度を高める |

### SVM（サポートベクターマシン）
| 用語 | 内容 |
|---|---|
| **マージン最大化** | 境界線からの距離（マージン）が最大になる分類線を求める考え方 |
| **サポートベクトル** | 境界線から一番近いデータ点 |
| **ハードマージン** | 誤分類を禁止してマージン最大化 |
| **ソフトマージン** | 誤分類を許容してマージン最大化（スラック変数で調整）|
| **スラック変数** | ソフトマージンで誤分類をどの程度許容するか調整する変数 |
| **カーネル法** | 次元を拡張させることで直線で分類できない問題に対応 |
| **カーネル関数** | 次元を拡張させる関数（カーネルトリックで計算を効率化）|

### K近傍法（KNN）
| 用語 | 内容 |
|---|---|
| 仕組み | 未知データに近いK個のデータを取得し、多数決でクラスを決定 |
| **Kが小さい** | 外れ値の影響を受けやすい（K=1で特に顕著）|
| **Kが大きい** | 精度が悪化しやすい |
| **次元の呪い** | 高次元になると距離の差が小さくなり精度が低下する問題 |
| **球面集中現象** | 高次元で距離の差が小さくなる数学的現象 |
| **Lazy Learner** | 学習フェーズがない（訓練データをそのまま保持する）|
| 注意点 | クラスのデータ量に偏りがあると精度低下。スケーリング必須 |

### 時系列分析
| モデル | 内容 |
|---|---|
| **ARモデル（自己回帰モデル）** | 過去のデータで現在の値を予測 |
| **VARモデル** | ARモデルを多変量に拡張 |
| **MAモデル（移動平均モデル）** | 過去の移動平均で現在の値を予測 |
| **ARMAモデル** | ARモデル＋MAモデルを組み合わせ |
| **ARIMAモデル** | 差分を取った時系列データにARMAを適用 |

## 🗂️ 教師なし学習

### クラスタリング
| 用語 | 内容 |
|---|---|
| **ハードクラスタリング** | 各データが1つのクラスタに属する |
| **ソフトクラスタリング** | 各データが複数のクラスタに属せる |
| **非階層クラスタリング** | 階層を作らずにクラスタに分類。k-means法が代表 |
| **階層クラスタリング** | 階層を作りながらクラスタ分け。デンドログラムで可視化 |

| アルゴリズム | 特徴 |
|---|---|
| **k-means法（非階層・ハード）** | あらかじめk個のクラスタ数を設定。重心を使って繰り返し更新 |
| **ウォード法（階層・ハード）** | 分散が最小になるようにデータをまとめる。デンドログラムで表示 |
| **群平均法（階層・ハード）** | クラスタ間の距離の平均を使ってデータをまとめる |

### 次元削減
| 手法 | 内容 |
|---|---|
| **PCA（主成分分析）** | データの分散が最大になる方向（主成分）に次元を削減 |
| **第1主成分** | ばらつきが最大の方向 |
| **第2主成分** | 第1主成分に垂直で次にばらつきが大きい方向 |
| **t-SNE法** | t分布を活用して次元を削減。可視化によく使用 |
| **特異値分解（SVD）** | 行列を分解して次元を削減。LSAに応用 |
| **多次元尺度構成法（MDS）** | データの類似度・距離を大小関係を維持して低次元に表現 |

### トピックモデル（文書分類）
| 手法 | 内容 |
|---|---|
| **LSA（潜在意味解析）** | 単語の頻度からカテゴリを推定。特異値分解を使用 |
| **PLSA** | LSAに確率の概念を組み合わせた手法 |
| **LDA（潜在的ディリクレ配分法）** | PLSAを発展。ディリクレ分布を使用 |

### レコメンデーション
| 手法 | 内容 |
|---|---|
| **協調フィルタリング** | 類似ユーザーが購入・視聴したものを推薦 |
| **コンテンツベースフィルタリング** | 商品の特徴を使って類似商品を推薦。コールドスタート問題を回避 |
| **コールドスタート問題** | 新規ユーザー・商品はデータ不足で適切な推薦ができない問題 |
| **アソシエーション分析** | 購入履歴などから関連性を見つける（オムツ＆ビールの法則など）|

## 🗂️ 強化学習

| 用語 | 内容 |
|---|---|
| **エージェント** | 自律的に行動するシステムやプログラム |
| **環境** | エージェントが行動する場所・状況 |
| **状態** | 現在の環境の状況 |
| **行動** | エージェントが選択するアクション |
| **報酬** | エージェントの行動を評価するスコア |
| **エピソード** | 行動の開始〜終了までの1回分の試行 |
| **割引率（γ）** | 未来の報酬を現在の価値に換算するハイパーパラメータ（0〜1）|
| **マルコフ性** | 状態遷移の確率が過去に依存せず現在の状態のみで決まる特性 |
| **マルコフ決定過程（MDP）** | マルコフ性を満たす遷移過程モデル |
| **活用（Exploitation）** | 既存情報から最適な行動を選択 |
| **探索（Exploration）** | 新しい情報を獲得するための行動 |
| **バンディットアルゴリズム** | 限られた資源で活用と探索の最適バランスを求めるアルゴリズム |

### 方策・価値関数
| 用語 | 内容 |
|---|---|
| **方策（Policy）** | エージェントの行動基準（活用と探索の割合）|
| **ε-greedy方策** | 基本は最適行動を選択するが、確率εで探索 |
| **UCB方策** | 探索時にあまり選択されていない行動を優先 |
| **行動価値関数（Q関数）** | ある状態で行動した後の将来の報酬合計を返す関数 |
| **状態価値関数（V関数）** | ある状態での将来の報酬合計を返す関数 |
| **Q値** | 行動価値関数の返り値。Q値最大の行動を選ぶ=Q学習 |
| **Q学習** | Q値を最大化するように学習。TD学習の一種 |
| **SARSA** | Q学習と似た手法（更新式が微妙に異なる）. TD学習の一種 |
| **TD学習** | エピソード終了を待たずに価値関数を更新する手法 |
| **価値反復法** | エージェントと環境のやり取りで価値関数を繰り返し更新 |
| **方策勾配法** | 方策をパラメータ関数として直接最適化 |
| **REINFORCE** | 方策勾配法の代表的なアルゴリズム（AlphaGoに使用）|
| **PPO** | REINFORCEを改良。パラメータ更新量を抑制して安定化 |
| **Actor-Critic** | 価値ベース（Critic）＋方策ベース（Actor）の組み合わせ |
| **A3C** | 複数エージェントが非同期・並列に学習するActor-Criticの応用 |

## 🗂️ モデルの評価手法

### 交差検証の種類

| 手法 | 内容 | 特徴 |
|---|---|---|
| **ホールドアウト検証** | データを一度だけ訓練データとテストデータに分割（7:3 や 8:2 が多い）| シンプル・高速。データが少ないと信頼性が低い |
| **k-分割交差検証（k-fold CV）** | データをk個に分割し、1個をテスト・残りを訓練として k回評価 | 信頼性高。計算コスト高 |
| **1つ抜き法（Leave One Out / LOO）** | 1件をテスト・残りを訓練 → 全件繰り返す | 最高精度だが計算コストが非常に高い。**ジャックナイフ法**とも |

- **訓練データ**：モデルの学習に使用
- **検証データ**：ハイパーパラメータ調整に使用
- **テストデータ**：最終的な性能評価に使用（学習中は使わない）

### データリーケージ
- 訓練データや検証データに、本来未知であるべきテストデータの情報が混入すること
- 時系列データでは**訓練データに未来のデータを含めてはいけない**
- データリーケージがあると本来の性能より高い評価が出てしまう

## 🗂️ 分類モデルの評価指標

### 混同行列
| | 陽性と判断 | 陰性と判断 |
|---|---|---|
| **正解が陽性** | **TP**（真陽性）| **FN**（偽陰性）= 第二種の過誤 |
| **正解が陰性** | **FP**（偽陽性）= 第一種の過誤 | **TN**（真陰性）|

### 主要指標
| 指標 | 計算式 | 重視する場面 |
|---|---|---|
| **正解率（Accuracy）** | (TP+TN)/(全数) | バランスが取れたデータ。クラスが極端に不均衡な場合は不向き |
| **適合率（Precision）** | TP/(TP+FP) | FPを減らしたい（スパムメール判定など）|
| **再現率（Recall）** | TP/(TP+FN) | FNを減らしたい（病気の診断など）|
| **F値（F1-score）** | 2×Precision×Recall / (Precision+Recall) | 適合率と再現率のバランス評価 |

- **適合率 ↑** → 曖昧なものを陰性にしがち → 再現率 ↓（トレードオフ）
- **再現率 ↑** → 曖昧なものを陽性にしがち → 適合率 ↓（トレードオフ）

### ROC曲線・AUC
| 用語 | 内容 |
|---|---|
| **真陽性率（TPR）** | TP ÷ (TP+FN) = 再現率 |
| **偽陽性率（FPR）** | FP ÷ (FP+TN) |
| **ROC曲線** | 縦軸TPR・横軸FPRで閾値変化をプロットした曲線 |
| **AUC** | ROC曲線の下の面積。1に近いほど性能が高い |
| AUC = 0.5 | ランダムな予測と同等 |

## 🗂️ 回帰モデルの評価指標
| 指標 | 特徴 |
|---|---|
| **MSE（平均二乗誤差）** | 誤差の2乗平均。外れ値に大きく影響される |
| **RMSE（二乗平均平方根誤差）** | MSEの平方根。単位が元データと同じ |
| **MAE（平均絶対誤差）** | 誤差の絶対値平均。外れ値に強い |
| **決定係数（R²）** | 0〜1。1が理想。相関係数の2乗。寄与率ともいう |
- RMSE > MAE：RMSE は大きな誤差を許容したくない場合に使用

## 🗂️ 過学習と正則化

### 過学習（オーバーフィッティング）
- **過学習**：訓練データへの過剰適応→未知データへの精度低下
- **未学習**：正則化しすぎ・モデル単純すぎ→訓練データにも精度低
- **訓練誤差**：訓練データへの予測誤差（小さくなるが意味が薄い）
- **汎化誤差**：未知データへの予測誤差（小さくすることが目標）

### バイアス・バリアンス分解
| 用語 | 内容 |
|---|---|
| **バイアス** | モデルが単純すぎるために発生する誤差（未学習側）|
| **バリアンス** | モデルが複雑すぎるために発生する誤差（過学習側）|
| **ノイズ** | 減らすことができない誤差 |
| トレードオフ | バイアスを下げるとバリアンスが上がりやすい |

### 正則化手法
| 手法 | 内容 |
|---|---|
| **L0正則化** | 0でない重みの数でペナルティ。計算コスト高 |
| **L1正則化（Lasso）** | 重みの絶対値の総和でペナルティ。重みをゼロにして特徴選択効果 |
| **L2正則化（Ridge）** | 重みの2乗和でペナルティ。重みを小さく保ち滑らかなモデルに |
| **L1ノルム（マンハッタン距離）** | 要素の絶対値の総和 |
| **L2ノルム（ユークリッド距離）** | 要素の2乗和の平方根 |

### 過学習を防ぐその他のテクニック
`アンサンブル学習` `スパース化（特徴量を減らす）` `データ拡張（水増し）` `ドロップアウト` `早期終了（Early Stopping）` `バッチ正規化`

| 指標 | 特徴 |
|---|---|
| **AIC（赤池情報量規準）** | 複雑さと当てはまりのバランス。データ数の影響を受けにくい |
| **BIC（ベイズ情報量規準）** | データ数が大きいとシンプルなモデルが評価されやすい |
